# 软件架构

## 目标

- 评分结果确定、可测试、可解释。
- 保存时间点数据和模型版本，避免前视偏差。
- 免费数据源能够替换为未来的付费数据源。
- 当前版本不承担个股选择职责。

## 分层结构

```text
数据源适配器
    -> 不可修改的原始缓存
    -> 带时间点的标准化记录
    -> 公司与产业链板块映射
    -> 因子计算
    -> 板块聚合
    -> 不可修改的评分快照
    -> 本地HTTP接口 / SVG网页 / 导出 / 回测
```

## 当前实现

`v0.1.0`已经包含：

- JSON模型配置；
- 15个正式板块和3个观察板块；
- 确定性的产业质量评分；
- 研究状态判断；
- 合成演示快照读取；
- 零第三方运行依赖的本地HTTP服务器；
- 原生SVG气泡图及CSV导出。

真实数据适配器和时间点数据库属于下一阶段。

## 规划中的数据记录

- `sector`：稳定ID、名称、产业链层级、是否参与排名；
- `company`：证券代码、交易所和上市状态；
- `membership`：公司、板块、业务暴露、证据和生效时间；
- `financial_fact`：报告期、公告日、可用日、来源和数值；
- `forecast`：预测期、发布日期、机构数量和分歧程度；
- `moat_evidence`：围墙项目、评分、来源日期和复核日期；
- `sector_snapshot`：模型版本、截止日期、五维评分和置信度。

## 可复现性

正式快照不得被静默覆盖。数据修正或模型变更必须产生新版本，并保留旧快照及输入清单。
