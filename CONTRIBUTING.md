# 贡献指南

感谢关注“三高产业链板块评估模型”。本项目首先是一套可追溯的产业研究方法，其次才是可视化工具。

## 开发规则

1. 数据采集、标准化、评分、回测和网页展示必须分层。
2. 修改公式、权重、阈值或缺失值处理时，必须同步增加或修改测试。
3. 板块成分变化必须记录依据、生效日期和失效日期。
4. 不得提交密钥、付费数据、受限制的数据集或第三方完整研报。
5. 合成数据必须明确标记为演示数据，不得包装成真实市场结论。
6. 当前版本不加入个股选择、买卖价位和交易执行功能。

## 提交建议

提交信息建议使用清晰的动词开头，例如：

```text
feat: 增加板块历史轨迹
fix: 修正低置信度状态优先级
docs: 补充数据发布时间口径
test: 增加负利润场景测试
```

## Pull Request说明

请在PR中说明：

- 修改了什么研究假设；
- 影响哪些数据字段；
- 是否影响历史快照的可复现性；
- 运行了哪些测试；
- 是否引入新的数据授权或使用限制。
